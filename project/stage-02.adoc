= Stage 02: Проработка функционала проекта

На этом этапе прорабатывается функциональность, которую будет выполнять приложение. Нет никаких ограничений сверху по количеству функционала, но в то же самое время не стоит продумывать проект на месяцы разработки вперед.

NOTE: *MVP* - *Minimal Viable Product* (*минимально жизнеспособный продукт*)

image::mvp.jpg[MVP]

== Tasks

WARNING: Все нижеописанное, лучше делать с помощью *Mind Map*. В качестве online-инструмента может выступать например: https://miro.com

* Выделить CORE (ядро) вашего функционала, которое будет выступать как *MVP* для вашего проекта. Оно будет являться *EPIC 1*, с которого и начнется разработка проекта.

image::user-story-diagram.png[User story diagram]

* Сформулировать для этого *EPIC* *User Stories*, т.е. произвести декомпозицию глобальной задачи в пользовательский функционал. *User Story* должны соответствовать шаблону:

    КАК ${ROLE} ХОЧУ ${FUNCTIONALITY} ПОТОМУ ЧТО ${PROFIT}

** `${ROLE}` - роль пользователя в системе
** `${FUNCTIONALITY}` - функциональность, которая должна быть доступна этой пользовательской роли (некоторое ОДНО действие)
** `${PROFIT}` - польза или причина, по которой эта функциональность необходима.

* Выделить *Roles* возникшие в процессе формирования *User Story*
* Выделить еще несколько *EPICS* для будущей декомпозиции.
