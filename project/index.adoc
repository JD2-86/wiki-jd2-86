== Versioning

Для прозрачности понимания и использования следует придерживаться следующих принципов:

* Каждая *VERSION* продукта эквивалентен одному *EPIC*.
* Каждый последующие *EPIC* увеличивают *VERSION* продукта итеративно, т.е. `+1` к версии.
* *UML Class Diagram* должны составляться под каждую новую *VERSION* продукта, на базе предыдущей.
* *UML Class Diagrams* для прошлых *VERSIONS* должны храниться в первоначальном состоянии, а не измененными в рамках другой *VERSION*.
